# Nota técnica grupal EE3 - InkaRise

En InkaRise EE3 se mantiene el sistema de apuntado con el ángulo θ, calculado a partir de la dirección entre el jugador y el mouse. Para representar la dirección del disparo se usa el vector `(cos θ, sin θ)`, lo que permite mover los proyectiles en cualquier orientación del plano 2D. En esta versión se agregó la mecánica de ricochet, donde el proyectil cambia de dirección al tocar un muro usando la normal de colisión. La coherencia del rebote se valida aplicando la reflexión vectorial `direction.bounce(normal)`, verificando que el ángulo de salida cambie según la superficie impactada. También se integró un rayo tipo RayCast2D/equivalente llamado Rayo del Inti, que calcula una línea desde el jugador hasta un alcance máximo y revisa si algún objetivo está dentro de esa trayectoria. Si el rayo detecta un objetivo, se activa una respuesta de daño y se registra el resultado. Finalmente, se agregó un proyectil homing/seek que busca el objetivo más cercano. Este proyectil obtiene el ángulo deseado hacia el objetivo y corrige su dirección poco a poco con `lerp_angle`, controlado por el parámetro `turn_speed`. Para mantener estabilidad se limitaron valores como `max_bounces`, `ray_distance` y `turn_speed`, evitando datos atípicos que afecten la experiencia del jugador. Las tres mecánicas se integran en una misma escena para que el jugador pueda probar disparo normal, rebote, rayo y persecución sin cambiar de nivel. El HUD muestra variables y métricas como θ, normal, rebotes, alcance del rayo, turn_speed, intentos, impactos y tasa de acierto.
