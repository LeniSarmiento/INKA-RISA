# Nota técnica grupal - InkaRise v3

En esta versión del prototipo se mejoró InkaRise usando un personaje inca visual, un fondo 2D estilo videojuego y un sistema de niveles del 1 al 10. El disparo se basa en el ángulo θ, obtenido desde la posición del jugador hacia la posición del mouse. Con ese ángulo se calcula la dirección del proyectil usando coseno para el eje X y seno para el eje Y. Por eso, el proyectil puede salir en distintas direcciones sin programar cada trayectoria manualmente. También se implementó un patrón angular tipo spread, donde varios proyectiles se separan mediante Δθ. Si Δθ aumenta, el abanico de disparo se abre más; si disminuye, los proyectiles salen más juntos. Los niveles suben la dificultad aumentando la velocidad de los objetivos, reduciendo su tamaño, elevando su resistencia y aumentando la cantidad de objetivos activos. Además, el HUD muestra métricas como intentos, impactos, fallos y precisión. El juego realiza limpieza básica de datos corrigiendo valores atípicos, por ejemplo Δθ negativo, Δθ demasiado alto o cantidad par de proyectiles. De esta forma, el prototipo evidencia uso de trigonometría, análisis de datos y decisiones de balance orientadas a la experiencia del usuario.
